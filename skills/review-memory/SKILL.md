# Review Memory 관리

모든 프로젝트의 리뷰 기록을 **mrt 저장소 안 한 곳**에 모아 관리하는 스킬.
대상 프로젝트에는 아무 파일도 만들지 않으므로, 각 repo에서 따로 무시 설정을 할 필요가 없다.
이 메모리는 `pr-review`, `local-code-review`, `branch-review` 스킬이 자동으로 만들고 채운다.

## 메모리 구조 (중앙집중)

```
<mrt>/.review-memory/            # mrt의 .gitignore로 무시됨 → 커밋되지 않음
├── <project-slug>/              # 대상 프로젝트의 git 루트 basename
│   ├── .origin                  # 원본 프로젝트 절대경로 (목록/추적/충돌 식별용)
│   ├── conventions.md           # 반복 패턴/컨벤션 누적 (리뷰 스킬이 자동 관리)
│   ├── context.md               # 사용자가 직접 등록하는 사전 컨텍스트
│   └── reviews/
│       └── <날짜>-<대상>.md     # 예: 2026-06-12-pr-123.md, 2026-06-12-local-feat-login.md
├── mrt/ …                       # mrt 자신의 리뷰 기록도 동일 규칙으로 이곳에
└── <다른 프로젝트>/ …
```

- 경로 규칙은 전부 `scripts/lib.sh`(SSOT) 한 곳에 있다 — 위치를 바꾸려면 거기만 고친다.
- conventions.md는 리뷰 끝에 **사용자 확인을 거쳐 누적**되는 학습 파일(자동 기록하지 않는다), context.md는 사용자가 **미리 등록**하는 입력 파일이다. 둘 다 리뷰 시작 시 읽힌다.

## 통합 매니저

여러 프로젝트 메모리를 한 도구에서 다룬다. 인자는 **프로젝트 경로** 또는 **slug** 둘 다 받는다.

```bash
mrt-review <명령>
```

| 명령 | 설명 |
|------|------|
| `list` | 모든 프로젝트 메모리 나열 (slug, 리뷰 수, 최근 날짜, origin 경로) |
| `path <프로젝트\|slug>` | 메모리 폴더 절대경로 출력 |
| `show <slug>` | conventions.md + 최근 리뷰 목록 요약 |
| `setup <프로젝트경로>` | 메모리 폴더 보장 생성 |
| `export <프로젝트\|slug> [출력dir]` | 한 프로젝트 메모리를 tar.gz로 내보내기 |
| `import <아카이브> <프로젝트\|slug>` | 아카이브를 대상 메모리로 병합 |
| `rm <slug> --yes` | 해당 프로젝트 메모리 삭제 (확인 인자 필수) |

조회성 요청("리뷰 기록 보여줘", "어떤 프로젝트들이 있지?", "자주 나온 이슈가 뭐였지?")은
`mrt-review list` / `mrt-review show <slug>` 결과를 읽고 요약해 답한다.

## 작업별 방법

모든 스크립트는 이 스킬의 `scripts/` 폴더에 있고, 본문은 PATH 런처 `mrt-review`로 호출한다
(런처도 같은 `scripts/mrt-review`, 공통 경로 코어는 `scripts/lib.sh`).

### 초기화 / 폴더 보장

```bash
mrt-review setup <프로젝트경로>
```

중앙 폴더 생성 + 템플릿(context.md/conventions.md) + `.origin` 기록까지 한 번에 처리한다. 이미 있으면 그대로 둔다.
출력 마지막 줄이 해당 프로젝트의 메모리 폴더 경로다.

### 리뷰 컨텍스트 등록 / 수정

사용자가 "리뷰할 때 X를 고려해줘", "이 프로젝트는 ~한 배경이 있어", "다음 리뷰부터 성능 위주로 봐줘"
처럼 앞으로의 리뷰에 반영할 정보를 주면:

1. 위 초기화 스크립트로 폴더를 보장한다 (context.md 템플릿이 함께 생성된다).
2. 출력된 메모리 경로의 `context.md`를 파일 읽기 도구로 읽고, 사용자가 준 내용을 알맞은 섹션(프로젝트 배경 /
   리뷰 시 중점 사항 / 팀 규칙)에 수정 도구로 추가·갱신한다. 템플릿의 괄호 안내문은 실제 내용으로 교체한다.
3. 저장 후 현재 context.md 전체를 사용자에게 보여주고 맞는지 확인한다.

"이번 리뷰에서만" 고려할 일회성 정보는 context.md에 쓰지 않는다 — 그 리뷰에서만 반영하고 버린다.
영구 등록은 사용자가 "앞으로", "항상", "등록해줘" 같이 지속성을 표현할 때만 한다.

### 내보내기 / 가져오기

```bash
mrt-review export <프로젝트|slug> [출력디렉토리]
mrt-review import <아카이브.tar.gz> <프로젝트|slug>
```

export는 `<slug>-review-memory-<날짜>.tar.gz` 하나로 묶는다(백업·머신 이전·동료 공유).
import는 병합 방식이라 안전하다: 이미 있는 리뷰 파일은 건너뛰고, conventions.md는 새 내용만 뒤에 추가된다. 덮어쓰지 않는다.

## 주의

- 중앙 폴더(`<mrt>/.review-memory/`)는 mrt의 `.gitignore`로 무시된다. git에 직접 추가하지 않는다.
- 대상 프로젝트에는 메모리 파일을 만들지 않는다 — 각 repo의 `.gitignore`나 `.git/info/exclude`를 건드릴 필요가 없다.
- import 시 기존 기록을 덮어쓰지 말 것. 충돌이 의심되면 사용자에게 보여주고 결정을 맡긴다.
- `rm`은 되돌릴 수 없으므로 `--yes` 확인 인자가 있을 때만 실행한다.
